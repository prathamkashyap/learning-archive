# Issues, Discussions, and Projects

## Context

GitHub is not just for pushing code. Issues and Projects make the repository look maintained and help track what to improve next.

## Issues

Use issues for:

- Bugs.
- Documentation tasks.
- Feature ideas.
- Research notes.
- Cleanup work.

Issue title examples:

```text
Document Docker volume examples
Fix broken Markdown links
Add SQL index notes
```

## Discussions

Use discussions for open-ended questions:

- Which topic should be expanded next?
- How should notes be structured?
- What examples are missing?

## Projects

Project columns:

```text
Backlog -> Ready -> In Progress -> Review -> Done
```

## Common Mistakes

- Using issues as vague reminders with no next action.
- Creating too many labels too early.
- Letting completed tasks stay open.
- Mixing discussion questions with actionable bug reports.

## Quick Reference

| Need | GitHub feature |
| --- | --- |
| Track a task | Issue |
| Open-ended conversation | Discussion |
| Plan across issues | Project |
| Group issues | Labels |
| Assign responsibility | Assignees |
