# GitHub Profile

## Context

The profile is often the first page someone sees before they open any repository. I do not want it to look overdesigned, but I do want it to clearly show what I am learning, what I can build, and which repositories are worth opening.

## Commands / Steps

Create a special profile repository with the same name as the GitHub username:

```text
username/username
```

Add a `README.md` inside that repository. Useful sections:

- Short introduction.
- Current learning focus.
- Featured repositories.
- Tools and languages.
- Contact or portfolio links.

Pin useful repositories from the GitHub profile page so visitors see the strongest work first.

## Common Mistakes

- Adding too many badges and making the profile noisy.
- Pinning unfinished repos without a clear README.
- Writing a generic bio that does not explain current direction.
- Letting old learning claims stay outdated.

## Quick Reference

| Task | Where |
| --- | --- |
| Profile README | `username/username` repository |
| Pinned repos | GitHub profile page |
| Public learning proof | Clear README files and commit history |
| Keep profile current | Review monthly |
