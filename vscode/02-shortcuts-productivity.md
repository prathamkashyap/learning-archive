# VS Code Shortcuts and Productivity

## Context

The fastest VS Code habit is using search and the command palette instead of hunting through menus.

## macOS Shortcuts

| Action | Shortcut |
| --- | --- |
| Command palette | `Cmd+Shift+P` |
| Quick open file | `Cmd+P` |
| Search in files | `Cmd+Shift+F` |
| Toggle terminal | Ctrl+backtick |
| Format document | `Shift+Option+F` |
| Multi-cursor | `Option+Click` |
| Rename symbol | `F2` |
| Go to definition | `F12` |

## Search

Useful search patterns:

```text
TODO
functionName
*.md
src/**/*.ts
```

## Refactoring

| Task | Shortcut/command |
| --- | --- |
| Rename symbol | `F2` |
| Quick fix | `Cmd+.` |
| Go to references | `Shift+F12` |
| Organize imports | Command Palette |

## Terminal

```bash
pwd
git status --short
npm test
docker compose up -d
```

## Common Mistakes

- Searching the whole repo when a folder filter would be faster.
- Editing repeated text manually instead of using multi-cursor.
- Forgetting the command palette can find almost every VS Code action.
- Using mouse-only navigation for common tasks.
