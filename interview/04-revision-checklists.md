# Revision Checklists

## Context

This is the fast skim list before interviews. It points me back to the sections in this repo instead of making a second copy of all notes.

## Technical Revision

| Area | Check |
| --- | --- |
| Git | Branches, merge vs rebase, undo commands |
| GitHub | Pull requests, issues, actions, branch protection |
| Linux | Permissions, processes, `grep`, `find`, SSH |
| Docker | Images, containers, volumes, Compose |
| SQL | Joins, aggregation, indexes, transactions |
| Networking | HTTP, DNS, TCP vs UDP, ports, TLS |
| System Design | Caching, queues, databases, load balancing |

## Before Interview

- Prepare two project explanations.
- Prepare five STAR stories.
- Review common commands.
- Revisit mistakes from previous practice.
- Keep resume projects consistent with GitHub READMEs.

## Questions to Ask Interviewer

- What does success look like in this role?
- What kind of projects would I work on first?
- How does the team review code?
- What is the current technical challenge for the team?

## Common Mistakes

- Revising only algorithms and ignoring projects.
- Forgetting behavioral examples.
- Not knowing what is on the resume.
- Asking no questions at the end.

## Quick Reference

| Time left | Focus |
| --- | --- |
| 1 week | Projects, SQL, Git, system design basics |
| 1 day | STAR stories, project summary, command cheatsheets |
| 1 hour | Resume, top mistakes, questions to ask |
